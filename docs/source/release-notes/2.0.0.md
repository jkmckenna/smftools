(v2.0.0)=

2.0.0 2026-07-12
Version 2.0.0 split `smftools experiment load` into separate raw-ingestion and dense-cache stages, replacing the 1.x partitioned-store design. Raw ingestion became ragged and partitioned, reads are linked back to their origin POD5 file with per-read sequencing/signal metadata, and per-base current-signal features are derived from the move table into densifiable layers.
