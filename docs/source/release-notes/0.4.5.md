(v0.4.5)=

0.4.5 2026-07-12
Version 0.4.5 removed a wasteful disk round-trip in `smftools experiment load`'s sequence-encoding step: previously each batch of integer-encoded read sequences was written to a throwaway `.h5ad` file and immediately read back in the same worker process, with no memory benefit. Sequences are now encoded directly into an in-memory mapping, with a new unit test covering the encoding path.
