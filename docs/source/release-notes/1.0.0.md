(v1.0.0)=

1.0.0 2026-07-12
Version 1.0.0 introduced the partitioned-store architecture: experiment data is now written as zarr partitions (keyed by reference x sample) plus a thin `spine.h5ad` molecule index and a catalog, replacing the single monolithic AnnData file. Added zarr-safe I/O (reusing the h5ad sanitizer), made `smftools experiment load` additively emit the partitioned store, and added a `migrate-store` CLI command plus store paths on `AdataPaths`. This design was superseded in 2.0.0 by the raw/load split and in 2.1.0 by the project-level catalog, but established the on-disk partitioning model both build on.
